# ASTRO Heatmap — Secondary Screen Design Document
## 二次スクリーン設計仕様書

### 1. 概要

一次スクリーン（choroplethヒートマップ）でTotal Score ≥ 56の152自治体を特定。
二次スクリーンでは、これらの自治体をゾーンごとに深掘りし、
不動産投資判断に必要な**需要持続性**と**リスク**を可視化する。

### 3. 二次スクリーンのレイヤー構成（優先順位順）

#### Layer 1: 立地適正化計画 ★最優先
- **API**: `XKT003` (都市計画決定GIS 立地適正化計画区域)
- **データ**: 居住誘導区域、都市機能誘導区域、立地適正化計画区域
- **投資意義**: 行政の「ここに人を集めたい」意思＝将来の公共投資・インフラ維持方向
- **表示**: ポリゴンオーバーレイ（居住誘導=緑、都市機能=青、区域外=グレー）
- **スコアリング**: 物件候補地が居住誘導区域内かどうか → Pass/Fail

#### Layer 2: 将来推計人口（年齢構成）
- **API**: `XKT013` (将来推計人口 250mメッシュ)
- **データ**: 総人口、年齢階級別人口、年齢構成比率（2020→2050）
- **投資意義**: 「人口減少かどうか」でなく「子育て層が残るか、高齢化だけ進むか」
- **表示**: 250mメッシュのコロプレス（若年層比率の変化）
- **スコアリング**: 
  - 20-39歳比率2050年推計 > 20% → High
  - 15-25% → Medium  
  - < 15% → Low

#### Layer 3: 生活インフラ（教育・保育・医療）
- **API**: 
  - `XKT006` (学校)
  - `XKT007` (保育園・幼稚園)  
  - `XKT008` (医療機関)
- **データ**: 施設名、所在地、分類（病院/診療所、病床数、救急告示等）
- **投資意義**: ファミリー実需 vs 高齢者需要の切り分け
- **表示**: ポイントマーカー（種別で色分け）
- **スコアリング**: 
  - 駅800m圏内の施設密度
  - 小学校徒歩10分圏内 → ファミリー適性High

#### Layer 4: 都市計画規制
- **API**: 
  - `XKT004` (用途地域)
  - `XKT005` (防火・準防火地域)
  - 都市計画道路（API TBD）
  - 地区計画（API TBD）
  - 高度利用地区（API TBD）
- **データ**: 用途地域区分、容積率、建ぺい率、防火指定
- **投資意義**: 開発余地、建替えコスト、ボリューム感
- **表示**: ポリゴンオーバーレイ（用途地域色分け）

#### Layer 5: 防災（ハザード）
- **API**: 
  - `XKT009` (洪水浸水想定区域)
  - `XKT010` (土砂災害警戒区域)
  - 液状化（API TBD）
  - 大規模盛土造成地（API TBD）
- **データ**: 浸水深ランク、土砂災害区分
- **投資意義**: リスク調整後のスコアリング
- **表示**: ポリゴンオーバーレイ（浸水深で色分け）
- **スコアリング**: 
  - 浸水想定0.5m以上 → Risk flag
  - 土砂災害警戒区域内 → Risk flag
  - 液状化リスク高 → Risk flag

#### Layer 6: 駅勢力圏・乗降客数
- **API**: `XKT015` (駅別乗降客数)
- **データ**: 駅名、路線名、各年乗降客数
- **投資意義**: 「売る相手が残るか」= 出口の広さ
- **表示**: 駅ポイント＋800mバッファ＋乗降客数ラベル
- **スコアリング**: 
  - 乗降客数 > 50,000/日 → High
  - 10,000-50,000 → Medium
  - < 10,000 → Low

### 5. API取得戦略

不動産情報ライブラリAPIはXYZタイル座標で指定。
152自治体をカバーするには、各自治体のbounding boxから
必要なタイル座標を計算し、一括取得→ローカルキャッシュする。

```
取得フロー:
1. zones.json から自治体リストを読み込み
2. 各自治体の座標からz=14のタイル範囲を計算
3. API別にタイルを順次取得（レート制限に配慮）
4. GeoJSONをcache/に保存
5. screen.htmlがcache/から読み込んで表示
```

#### タイル数の見積もり
- z=14: 1タイル ≈ 1km × 1km
- 152自治体の面積合計 ≈ 推定3,000km²
- API 6種 × 3,000タイル ≈ 18,000リクエスト
- レート制限を考慮: 1req/sec → 約5時間
- 実際は重複タイルが多いので ≈ 2-3時間

### 8. 技術選定

- **地図**: MapLibre GL JS（無料、ベクトルタイル対応、GeoJSON直接読み込み可）
- **背景地図**: 地理院タイル（無料、高品質）
- **データ取得**: Python (urllib, gzip対応)
- **表示**: 単一HTML（一次スクリーンと同じアーキテクチャ）
- **キャッシュ**: ローカルJSONファイル（GitHub Pagesで静的配信可能）
